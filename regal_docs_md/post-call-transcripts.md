# Post-Call Transcripts

See [this article for Conversation Intelligence feature](https://support.regal.ai/hc/en-us/articles/17259619778331-Reviewing-Call-Recordings-Transcripts)details, and where to see post-call transcripts in app.

This guide below walks through what additional information users can observe and monitor for AI agent calls.

## AI Actions

AI Actions i.e. function calls made by the AI agent is additionally displayed along with transcript texts to help you understand what happens behind the utterances, whether agent took an action, and what the action result is. These information often dictates why agent say certain things.

![](https://files.readme.io/c7c26106e07981e99176358834f242fcbc7a6f4e2610fb90341d0d22e5963f68-image.png)

Note: Knowledge Base queries also show up as function calls with below name Action Invocation: knowledge\_base\_name

## Action Errors

If any of the agent actions resulted in an error state, they will be displayed as a call level warning. Upon hitting "View", you will be navigated to the failed action in the transcript (marked red). Take quick action to debug and reconfigure the agent actions.

![](https://files.readme.io/f4315e4d13c9189365f0df0ac4f10bc453963e93bb0a70070bb5e66321404910-image.png)

  

## Multi-state State Transitions

For multi-state agents, when the agent transition from one state to the next will show up along the conversation.
